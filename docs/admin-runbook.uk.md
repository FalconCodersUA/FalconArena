# Runbook для адміністратора

## Призначення

Цей runbook описує щоденний сценарій роботи `ADMIN` у FalconArena:

- підготовка турніру
- запуск реєстрації
- створення раунду
- контроль сабмітів і оцінювання
- робота з інтеграціями
- завершення турніру і архів

Документ орієнтований на реальну експлуатацію платформи під час пілоту або демо.

## 1. Перед початком роботи

Переконайтесь, що:

- ви увійшли під `ADMIN`
- `/app/admin` відкривається без помилок
- `/app/integrations` відкривається і ключові сервіси в зрозумілому стані
- якщо планується показ інтеграцій, `Google Sheets` та `Email delivery` вже налаштовані

## 2. Створення турніру

Маршрут:

- `/app/admin`

Що зробити:

1. Натиснути `Створити турнір`.
2. Заповнити:
 - назву
 - опис
 - старт турніру
 - вікно реєстрації
 - max teams, якщо потрібно
3. Зберегти турнір.

Що перевірити:

- турнір з'явився у списку
- статус стартово коректний
- публічна сторінка `/app/tournaments/:id` відкривається

## 3. Відкриття реєстрації

Маршрут:

- `/app/admin`

Що зробити:

1. Вибрати турнір у блоці керування.
2. Змінити статус на `REGISTRATION`.

Що перевірити:

- на публічній сторінці турніру видно стан реєстрації
- `TEAM` може зареєструвати команду у дозволеному вікні
- bell / повідомлення показують системне сповіщення про старт реєстрації

## 4. Створення раунду

Маршрут:

- `/app/admin`

Що зробити:

1. Натиснути `Створити раунд`.
2. Заповнити:
 - назву
 - опис
 - must-have
 - вимоги до технологій
 - додаткові матеріали
 - старт
 - дедлайн
3. Зберегти раунд.
4. За потреби перевести його в активний статус.

Що перевірити:

- раунд видно в адмінці
- раунд видно на публічній сторінці турніру
- `TEAM` бачить активний раунд у своєму кабінеті

## 5. Розклад турніру

Маршрути:

- `/app/admin`
- `/app/tournaments/:id`

Що зробити:

1. Додати ключові події:
 - старт реєстрації
 - консультація
 - старт раунду
 - дедлайн
 - оголошення результатів
2. За потреби редагувати або видаляти події.

Що перевірити:

- події видно на публічній сторінці турніру
- події видно в `TEAM` і `JURY` кабінетах

## 6. Користувачі та ролі

Маршрут:

- `/app/admin`

Що зробити:

1. Створити користувачів `JURY` або `ORGANIZER`, якщо вони потрібні.
2. Не створювати зайвих `ADMIN`, якщо це не потрібно операційно.

Що перевірити:

- новий акаунт може увійти
- роль відкриває правильний кабінет

## 7. Контроль сабмітів

Маршрути:

- `/app/team`
- `/app/admin`
- `/app/tournaments/:id`

Що перевірити:

- команда може зберегти або оновити сабміт до дедлайну
- після дедлайну сабміт блокується
- у повідомленнях та системних сповіщеннях з'являються відповідні події

## 8. Розподіл оцінювання

Маршрут:

- `/app/admin`

Що зробити:

1. Переконатися, що є користувачі `JURY`.
2. Запустити `розподіл призначень`.
3. Перевірити кількість reviewer-ів на submission.

Що перевірити:

- `JURY` бачить призначені роботи у `/app/jury`
- оцінювання зберігається
- `activity feed` показує ключові дії

## 9. Завершення оцінювання і результати

Маршрути:

- `/app/admin`
- `/app/leaderboard`
- `/app/archive`

Що зробити:

1. Завершити оцінювання для раунду.
2. Перевірити leaderboard.
3. За потреби експортувати:
 - CSV
 - Google Sheets

Що перевірити:

- у `Інтеграціях` видно останній status export
- `Archive` показує фінальні результати

## 10. Сертифікати

Маршрут:

- `/app/archive`

Що зробити:

1. Відкрити завершений турнір.
2. Заповнити або відредагувати шаблон сертифіката.
3. Відкрити:
 - `Сертифікат участі`
 - `Сертифікат переможця`

Що перевірити:

- printable preview відкривається
- браузерний `Print / Save as PDF` працює

## 11. Інтеграції

Маршрут:

- `/app/integrations`

Що перевірити:

### Google Sheets

- webhook URL збережений
- `Test connection` успішний
- `Last export` оновлюється після реального експорту leaderboard

### Email delivery

- sender / provider / API key збережені
- `Send test email` працює
- `Last delivery` і message оновлюються

### Notification rules

- toggle-правила відповідають бажаній поведінці

### Tournament defaults

- нові турніри і раунди стартують з правильними дефолтами

## 12. Що перевіряти перед демо

1. `ADMIN` логін
2. `/app/admin`
3. `/app/integrations`
4. один активний турнір
5. один активний раунд
6. хоча б один `TEAM` сабміт
7. хоча б один `JURY` assignment
8. `/app/leaderboard`
9. `/app/archive`
10. сертифікати

## 13. Коли ескалювати технічну проблему

Потрібно залучати технічний розбір, якщо:

- `/health` не відповідає
- міграції не застосовуються
- login повертає `500`
- Google Sheets export або email test стабільно падають
- сторінки `admin / leaderboard / archive / profile` дають runtime error

У таких випадках переходьте до [ops-runbook.uk.md](/D:/MixProjects/FalconArena/docs/ops-runbook.uk.md) і `support/troubleshooting` документа.
