# UI Audit та Redesign Roadmap

## Мета

Перевести FalconArena з функціонального MVP у більш цілісний сучасний продукт для демо, захисту та подальшого розвитку без зміни основного бізнес-функціоналу.

## Короткий аудит поточного стану

### 1. Shell та загальна система

- Базова візуальна система вже зібрана: є токени кольорів, радіуси, тіні, єдина auth-стилістика та app-shell.
- Проблема не в повній відсутності стилю, а в неоднорідній ієрархії між екранами.
- Частина сторінок уже виглядає цілісно, але `Admin Dashboard` та `Інтеграції` все ще більше схожі на стек робочих блоків, ніж на сучасний control center.

### 2. Admin Dashboard

- Сильна сторона: функціонально екран уже насичений і корисний.
- Слабка сторона: перший екран не формує чіткий пріоритет дій для адміна.
- Бракує верхнього шару з:
  - поточним контекстом
  - onboarding / next steps
  - швидким доступом до ключових admin tools

### 3. Інтеграції / Системні налаштування

- Функціонально сторінка вже дуже сильна.
- Візуально вона поки виглядає як кілька великих форм одна під одною.
- Потрібно перетворити її на `admin toolset`:
  - короткий summary
  - статуси ключових систем
  - швидка навігація по секціях
  - акцент на operational clarity

### 4. Empty / loading / error states

- Частина станів уже уніфікована, але ще не на всіх сторінках вони однаково “дизайново” виглядають.
- Для захисту важливо, щоб навіть порожні стани виглядали навмисно, а не як технічний fallback.

### 5. Mobile UX

- Основні екрани вже адаптуються.
- Далі важливо не просто “щоб влізало”, а щоб мобільний інтерфейс зберігав ієрархію:
  - менше перевантажених first screens
  - компактні CTA
  - зручна навігація в tools / forms / messages

## Що вже варто вважати зробленим

- auth-екрани приведено до спільної стилістики
- app-shell уніфіковано
- ключові стани на `messages / archive / leaderboard / profile` уже частково приведені до спільного патерну
- мобільний pass для основних користувацьких сторінок уже виконано

## План редизайну

### Фаза 1. Admin Control Center

Ціль: зробити адмінку більш сучасною, керованою та зрозумілою з першого екрана.

Що робимо:

- верхній `workspace / control center` блок
- onboarding checklist для адміністратора
- admin toolset з переходами в:
  - інтеграції
  - повідомлення
  - лідерборд
  - архів
- сильніша ієрархія між quick actions, оглядом і керуванням турніром

### Фаза 2. Integrations as Admin Toolset

Ціль: перетворити системні налаштування в сучасний operational screen, а не просто в набір форм.

Що робимо:

- summary-card зверху
- cards для Google Sheets / Email / Tournament defaults / Notification rules
- швидка навігація по секціях
- зрозумілі operational statuses

### Фаза 3. Shared Design System Tightening

Ціль: добити єдину візуальну систему на всіх робочих екранах.

Що робимо:

- єдині layout-ритми
- єдині admin helper blocks
- єдині tool cards
- єдині secondary callouts
- ще сильніша типографічна ієрархія для section headers та card headers

### Фаза 4. Final Product Polish

Ціль: довести проект до стану “готовий до показу без незручних місць”.

Що робимо:

- фінальний ручний проход по всіх ролях
- точкові правки mobile UX
- додаткова чистка форм і CTA
- виправлення виявлених дрібних неузгодженостей

## Що варто покращити далі після цього проходу

### Onboarding / підказки для admin

- короткі helper-описи на складних етапах
- перший-run checklist
- contextual guidance для статусів турніру, раундів та інтеграцій

### Empty states

- більш виразні порожні стани з CTA
- спеціальні empty states для:
  - archive
  - leaderboard
  - messages
  - system integrations

### System changelog / activity feed

- окремий operational feed для admin:
  - хто створив турнір
  - хто змінив статус
  - хто створив round
  - хто оновив інтеграції
  - хто зберіг шаблон сертифіката

Це не блокер для MVP, але сильно підсилює відчуття “живої системи”.

### Better search

- пошук не лише по navigation sections, а по:
  - турнірах
  - командах
  - діалогах
  - оголошеннях

### Full admin toolset

- оформити інтеграції, експорт, архів і operational controls як повноцінний набір admin tools
- у перспективі можна винести це в окрему “операційну” навігацію або dashboard widgets

## Пріоритет

1. `Admin Dashboard + Integrations redesign`
2. `shared visual system tightening`
3. `final demo polish`
4. `activity feed / better search` як наступний етап після захисту або якщо залишиться час
